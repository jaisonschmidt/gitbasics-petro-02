# Introdução ao Git

Git é um sistema de controle de versão distribuído amplamente utilizado no desenvolvimento de software. Ele permite que desenvolvedores rastreiem alterações no código, colaborem em projetos e gerenciem diferentes versões de um repositório de forma eficiente.

## Principais Recursos do Git

- **Controle de Versão**: Permite registrar o histórico de alterações em arquivos.
- **Branching e Merging**: Facilita o trabalho em diferentes funcionalidades simultaneamente e a integração de mudanças.
- **Distribuído**: Cada desenvolvedor possui uma cópia completa do repositório.
- **Desempenho**: Operações locais são rápidas e eficientes.

## Comandos Básicos do Git

- `git init`: Inicializa um novo repositório Git.
- `git clone`: Clona um repositório existente.
- `git add`: Adiciona arquivos ao índice (staging area).
- `git commit`: Salva as alterações no histórico do repositório.
- `git push`: Envia alterações para um repositório remoto.
- `git pull`: Atualiza o repositório local com mudanças do remoto.

## Benefícios do Git

- **Colaboração**: Facilita o trabalho em equipe.
- **Rastreamento**: Mantém um histórico detalhado das alterações.
- **Flexibilidade**: Suporte para fluxos de trabalho variados.

Git é uma ferramenta essencial para qualquer desenvolvedor, promovendo organização e eficiência no desenvolvimento de software.
